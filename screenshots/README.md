# Screenshots Directory

This directory contains screenshots for the main README.md file.

## Required Screenshots

### iOS Screenshots (iPhone)
Place these screenshots in this directory:

- `ios-chat-list.png` - Main chat list view showing conversations
- `ios-chat-view.png` - Individual chat conversation with messages
- `ios-qr-code.png` - My QR Code view for sharing contact
- `ios-scan-qr.png` - QR code scanner view
- `ios-contacts.png` - All contacts list view
- `ios-settings.png` - Settings view

**Recommended Size**: 1170 x 2532 pixels (iPhone 14 Pro Max)

### macOS Screenshots
Place these screenshots in this directory:

- `macos-main-window.png` - Main window showing sidebar and chat view
- `macos-qr-modal.png` - QR code modal window
- `macos-contacts-modal.png` - Contacts modal window

**Recommended Size**: 2880 x 1800 pixels or larger (Retina resolution)

### Optional
- `beam-logo.png` - App logo/icon (512x512 pixels minimum)

## How to Take Screenshots

### iOS
1. Run the app in the iOS Simulator or on a real device
2. Press `Cmd + S` (Simulator) or Side Button + Volume Up (iPhone)
3. Screenshots will be saved to Desktop or Photos

### macOS
1. Run the app on macOS
2. Press `Cmd + Shift + 4` then press `Space` to capture window
3. Click on the window to capture
4. Screenshot will be saved to Desktop

## Tips for Great Screenshots

- Use light mode for consistency (or show both light/dark)
- Include some sample data (contacts, messages)
- Make sure UI elements are visible and clear
- Use high-resolution displays if possible
- Consider adding device frames using tools like:
  - [AppLaunchpad](https://theapplaunchpad.com/)
  - [Screely](https://www.screely.com/)
  - [Device Frames](https://deviceframes.com/)

## Adding Screenshots to README

Once you've added the screenshots to this directory, they will automatically be referenced in the main README.md file using relative paths like:

```markdown
<img src="screenshots/ios-chat-list.png" alt="iOS Chat List" width="250"/>
```

Make sure to commit and push these images to your GitHub repository for them to display properly on GitHub.
